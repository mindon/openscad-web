# OpenSCAD Playground - Refactored Architecture

## Architecture Overview

The project uses a **flat-file backend + Lit frontend** architecture, served by
a single `biu --serve` command:

```
openscad-playground/
├── backend/              # Next.js style Backend API (flat .ts route files)
│   ├── api/              # API endpoints
│   │   ├── check-syntax.ts
│   │   ├── export-3mf.ts
│   │   ├── export-glb.ts
│   │   ├── files/
│   │   │   └── [fileId].ts
│   │   ├── health.ts
│   │   └── render.ts
│   └── _lib/             # Shared utilities
│
├── src/                  # Frontend (Lit Web Components + biu)
│   ├── components/       # Lit components (*.ts)
│   ├── state/            # State management
│   ├── config.ts         # API endpoint config
│   └── index.ts          # Entry point
├── static/               # Static assets
└── dist/                 # Build output (generated by biu)
```

The server dynamically imports route files based on HTTP method and URL path,
with no build step or route registration needed.


## Frontend (Lit + biu)

- **Framework**: Lit 3.0 (Web Components, no TSX)
- **Serve tool**: `biu --serve` (zero-config bundler powered by Bun)
- **Editor**: Monaco Editor (loaded via CDN)
- **3D Viewer**: model-viewer (`<model-viewer>`)

### Prerequisites

- `biu` requires [Bun](https://bun.sh/) runtime:

- get `biu` from <https://github.com/mindon/biu>


**wasm and libraries:**

**static/wams/** and **static/runner/libraries** clone repo from <https://github.com/openscad/openscad-playground>

to the root directory of the clone repo:

run `bun run build:libs:wasm`  to build wasm and copy from **public/openscad.*** to **static/wasm/** of this project.

run `bun run build:libs` to build libraries and copy from **public/libraries** to **static/runner/libraries** of this project.


### Serve

```bash
# Start frontend dev server (biu, port 3000)
biu --serve
```

## Backend

- **API Endpoints**:
  - `GET /api/health` - Health check
  - `POST /api/render` - Render OpenSCAD source
  - `POST /api/check-syntax` - Check OpenSCAD syntax
  - `POST /api/export-3mf` - Export as 3MF
  - `POST /api/export-glb` - Export as GLB
  - `GET /api/files/:fileId` - Download generated file

### API Requirements

The backend requires `openscad` CLI to be installed and available in `PATH`.

Install OpenSCAD:

- Download from https://openscad.org/
- create a `openscad` CLI in `PATH` linked with the `OpenSCAD`

**Zero-config server** - Uses `biu --serve` to serve both the API and frontend.

## Project Structure

### Frontend (`src/`)

| File                             | Description                                      |
| -------------------------------- | ------------------------------------------------ |
| `index.ts`                       | Entry point, mounts `<openscad-app>`             |
| `config.ts`                      | API base URL configuration                       |
| `api-client.ts`                  | Fetch wrapper for backend API                    |
| `state/store.ts`                 | Simple state management (replaces React Context) |
| `state/types.ts`                 | TypeScript types                                 |
| `components/app.ts`              | Root `<openscad-app>` component                  |
| `components/editor-panel.ts`     | Monaco editor wrapper                            |
| `components/viewer-panel.ts`     | 3D/2D model viewer                               |
| `components/footer.ts`           | Status bar + action buttons                      |
| `components/customizer-panel.ts` | OpenSCAD parameter customizer                    |

### Backend API (`backend/api/`)

```
api/
├── check-syntax.ts     # POST /api/check-syntax
├── export-3mf.ts       # POST /api/export-3mf
├── export-glb.ts       # POST /api/export-glb
├── files/
│   └── [fileId].ts     # GET /api/files/:fileId
├── health.ts           # GET /api/health
└── render.ts           # POST /api/render
```

Each API endpoint is a flat `.ts` file (not Next.js style `/route.ts`
subdirectories). The server dynamically imports these files based on the HTTP
method and URL path.
